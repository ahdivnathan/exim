Overview¶

After using the functions from clean.R to create instances of the data frames in this script, we have included the code we used to conduct our analysis on the data that we scraped and cleaned. There is a variety of analysis here, including basic linear regression, lagged regression, and cluster analysis, allowing us to see the trends that we describe in our project report IPython Notebook. After basic linear regression did not yield many fruitful results, we turned our focus to other analysis in order to fully understand the impact of trade diversification on a country's economic performance.

Now we will create new Data Frames that take slices of the trade Data Frame such that we can easily regress trade on our macroeconomic indicators. The new Data Frames will be specific to each indicator. This ensures that we do not have any countries in one data set that do not have corresponding data in the other data set

%%R
sliceTrade = function(df) {
  indCols = colnames(df)
  tradeCols = colnames(tradeIndex)
  commonCols = intersect(indCols, tradeCols)
  
  indRows = rownames(df)
  tradeRows = rownames(tradeIndex)
  commonRows = intersect(indRows, tradeRows)
  return(list(df[commonRows, commonCols], tradeIndex[commonRows, commonCols]))
}

indexSlice = sliceTrade(indexData)
co2Slice = sliceTrade(co2Emissions)
exportsSlice = sliceTrade(exportsGrowth)
gdpSlice = sliceTrade(gdpGrowth)
gdsSlice = sliceTrade(gdSaving)

Regression Analysis¶

We define the following function to regress the Trade Diversification Index of a country with one of its stock index, currency, or economic indicators.

In [6]: 
%%R
regressVectors = function(vectors) {
  depVariable = vectors[[1]]
  indVariable = vectors[[2]]
  fit = lm(depVariable ~ indVariable)
  summary(fit)
}

Basic linear regression does not show very clear results for certain variables when analyzed with TDI. Following more qualitative discussions, we have determined a very plausible reason. We are comparing the variable (e.g. Net Migration) with TDI in that same year. Keeping in mind that we are trying to study the effects of trade diversification, those effects may well be occuring, but will only be envident in the form of such metrics like Net Migration after some time.

Hence, we decided to lag certain variables with respect to TDI to make more meaningful statistical analyses. The following functions adjust the vectors accordingly. Depending on the variable in question, we investigated lagging the variable by up to 2 years, and lagging the TDI by up to 2 years.

prepVectors = function(slice) {
  indVector1 = as.vector(as.matrix(slice[[1]]))
  indVector1 = as.numeric(indVector1)
  tradeVector = as.vector(as.matrix(slice[[2]]))
  indVector2 = c()
  tradeVector2 = c()
  for (i in 1:length(indVector1)) {
    if (is.na(indVector1[i]) | is.na(tradeVector[i])) {
    }
    else {
      indVector2 = append(indVector2, indVector1[i])
      tradeVector2 = append(tradeVector2, tradeVector[i])
    }
  }
  return(list(indVector2, tradeVector2))
}

correlateVectors = function(vectors) {
  myMatrix = matrix(vectors[[1]])
  myMatrix = cbind(myMatrix, vectors[[2]])
  cor(myMatrix, use="all.obs", method="pearson")
}

K-means Clustering¶

The following functions generate the plots for k-means clustering.

%%R
clusterTrade = function() {
  tradeIndex = t(tradeIndex)
  tradeIndex = tradeIndex[rowSums(is.na(tradeIndex)) == 0,]
  clustered = kmeans(tradeIndex, 4)
  years = 1995:2011
  center1 = clustered$centers[1,]
  center2 = clustered$centers[2,]
  center3 = clustered$centers[3,]
  center4 = clustered$centers[4,]
  plot(years, center1, type="l", col="red", ylim=c(0, 700), ylab="Center Means", xlab="Years")
  par(new=TRUE)
  plot(years, center2, type="l", col="green", ylim=c(0, 700), ylab="Center Means", xlab="Years")
  par(new=TRUE)
  plot(years, center3, type="l", col="blue", ylim=c(0, 700), ylab="Center Means", xlab="Years")
  par(new=TRUE)
  plot(years, center4, type="l", col="cyan", ylim=c(0, 700), ylab="Center Means", xlab="Years")
  print(clustered)
}
